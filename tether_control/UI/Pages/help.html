Interface

This section controls how Tether Control talks to your C-Bus network. Control mode is for normal day-to-day operation, including controlling C-Bus from Home Assistant and updating entities from C-Bus network events. Interface mode provides a direct C-Bus connection for tools such as Toolkit and Commission.

The Enable, Disable and Reset buttons are for recovering from serial communication faults. If communication drops out, Reset is usually the best first step.

The SIM On and SIM Off buttons control the on-board relay used to disconnect and reconnect the C-Bus interface from the network. Use these to physically remove the interface from the network if the interface locks up.

Status

Status pills show current health at a glance. Green means working. Red means attention is needed.

Serial and SIM show communication and relay state. C-Bus Voltage helps confirm the bus is present and readable.

In Interface mode, the Client, Bytes and Last RX lines help confirm the connected tool and live traffic. Last error gives the most recent problem message if something fails.

C-Bus Applications

This area defines which C-Bus applications Tether Control should use and how they are named on the system.

Tether Control supports lighting applications 48 to 95 and the measurement application. Each enabled application can have a custom name.

Only applications enabled in this section will be functional.

MQTT Broker

This section connects Tether Control to your MQTT broker. Enter the broker address, port and any login details needed by your broker.

The status line confirms whether the connection is active. If disconnected, check address, port, username and password first.

Configured entities are published to Home Assistant under a single MQTT device named Tether Control. If your broker uses restrictions, make sure Tether Control is allowed to connect and publish discovery, state and command topics.

Tether Control publishes availability information so C-Bus entities become unavailable when control communications stop or Interface mode is active. Diagnostic entities for Mode, Serial Comms, SIM State and C-Bus Voltage remain available while Tether Control itself is connected to MQTT.

Network Time Manager

This section controls when time and date updates are sent to the C-Bus network for devices such as DLTs and eDLTs. Updates can be sent daily, at a repeating interval, or only when requested.

The time and date are taken from the host system. Use Set Now to send an immediate update after a network shutdown, device reconnection or clock correction. Scheduled updates inside the 11:50 PM to 12:10 AM no-go period are delayed until 12:11 AM.

C-Bus Group Setup

The C-Bus Group Setup page maps C-Bus groups and measurement channels to MQTT entities. These entities appear under the Tether Control device in Home Assistant.

Controls are output groups configured as a switch, light, dimmer or fan. The toggle provides quick on/off testing. Dimmers and fans also have a level control for setting the C-Bus level; the slider can be dragged or adjusted with the keyboard arrow keys.

Binary Sensors expose a C-Bus group state as an on/off sensor. Use Invert when the required sensor state is the opposite of the C-Bus group state.

Measurement Sensors map a C-Bus measurement unit and channel to current, temperature or humidity. Current measurements also create voltage, power and energy entities using the configured voltage.

Shutters map C-Bus groups to shutters, roller doors, blinds or curtains and support Up/Down/Off, Toggle and Pulse control methods. Their toggle button provides quick setup testing.

Entity Name becomes the Home Assistant entity name. Spaces are preserved in the displayed name and converted to underscores where Home Assistant requires an identifier-safe value.

Use Add for individual entries or Bulk Add to create a numbered group or channel range. Saving a group that uses an application which is not enabled will offer to add that application.

Use Import C-Bus Groups on the Tools page to merge entries from Toolkit XML or HTML project documents and compatible Node-RED flows JSON files. Import is experimental and may infer device types from names.

Click a column heading to sort a section. If configuration changes are waiting to be saved, Tether Control will ask whether to save or continue without saving before sorting or leaving the page.

Tools

The Tools page contains C-Bus group import and configuration transfer functions. C-Bus group imports are checked for existing address assignments and entity names before they are merged.

Configuration Export lets you select C-Bus Applications, C-Bus Group Setup, MQTT Broker, Network Time Manager and Interface sections individually or export them all.

Configuration Import reads a Tether Control export and lets you select which available sections to apply. C-Bus Group Setup entries can be merged with or replace the current list.

Saving Changes

Configuration changes are staged on screen until you press the relevant Save button. If you navigate away with unsaved configuration changes, Tether Control will offer to save them or continue without saving.

On the C-Bus Group Setup page, any Save button commits the complete current entity list. Live state changes made with a toggle or level slider are sent immediately and do not need to be saved.

Troubleshooting Basics

If something stops responding, first check the status pills and last error line. These usually point to the problem area quickly.

Confirm the relevant settings have been saved, use Reset in the Interface section, and check that power and network links are stable.

If issues continue, capture what changed just before the problem started. That makes remote support faster and more accurate.
